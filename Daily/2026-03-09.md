# 2. Main Technical Challenge

Dynamic compute allocation has a paradox:

```text
to know where to spend compute
you must spend compute
```

If routing cost > compute saved:

```text
dynamic system < static model
```

Solution used in practice:

Use **cheap proxies** instead of perfect evaluation.

Examples:

```text
prediction error
uncertainty
surprise
```

something to note: Value of Information (VoI)
$$\text{VoI} = \mathbb{E}_{z'}[V(z')] - V(z_t)$$

Your critique is **very solid**. You’ve identified a real failure mode that appears whenever a system uses **compression as the driver of compute allocation**. The tension between **entropy (information)** and **compressibility** is fundamental in representation learning, and if not separated carefully it absolutely can create the degeneracy you describe.

I’ll break this into three parts:

1. **Where your critique is exactly right**
2. **Where the failure can be prevented**
3. **How to formalize the corrected objective**

---

# 1. Where your critique is correct

### Compression ≠ importance

Your equation

[
I(z) = -\log p(z)
]

correctly states that **information corresponds to surprise**. Highly informative states deviate from the model’s prior.

If compute is allocated based on **compressibility**, the system optimizes for:

* low entropy
* high predictability
* low algorithmic complexity

That leads to a **predictability bias**.

The failure mode you describe is real:

**Compute starvation of rare but important states.**

This is a known issue in several systems:

* early **predictive coding models**
* naive **variational world models**
* some **compression-based intrinsic motivation systems**

The system becomes **comfort-seeking**.

---

# 2. Where the system collapses (your key insight)

You also correctly identified the **rollout degeneracy**.

If the objective indirectly rewards compression, the transition model can collapse into:

[
T(z_{t+1}|z_t,a_t) \rightarrow z^*
]

where (z^*) is a **stable attractor**.

This produces:

* trivial rollouts
* perfect predictability
* catastrophic real-world performance

This is analogous to:

* **posterior collapse in VAEs**
* **mode collapse in GANs**
* **reward hacking in RL**

Your observation is basically:

> The world model can cheat by simplifying reality.

Correct.

---

# 3. The missing piece: Expected Value of Compute (EVC)

Your proposed fix is also aligned with **optimal metareasoning theory**.

Instead of allocating compute via compression, you allocate compute via:

[
\text{EVC} = \text{Expected Value of Computation}
]

Classic formulation (Russell & Wefald):

[
\text{EVC} = \mathbb{E}[U(\pi_{\text{after compute}})] - U(\pi_{\text{current}}) - C
]

Where:

* (U) = utility / value function
* (C) = compute cost

You suggested:

[
\text{VoI} = \mathbb{E}_{z'}[V(z')] - V(z_t)
]

This is essentially a **Value of Information estimator**, which is a good proxy.

---

# 4. Correct architectural separation

The clean fix is exactly what you suggested:

### Two independent gradients

**1. Compression gradient (representation learning)**
Used only to build the latent hierarchy.

Example:

[
\mathcal{L}_{world} =
\mathbb{E}[-\log p(x|z)] +
\beta KL(q(z|x)||p(z))
]

This builds the **map**.

---

**2. Utility gradient (compute allocation)**
Controls **thinking depth**.

Example:

[
\mathcal{L}*{compute} =
C_r \cdot N*{steps}
-------------------

\lambda \cdot \Delta V
]

where

[
\Delta V =
V_{\text{after rollout}} - V_{\text{before rollout}}
]

This allocates the **fuel**.

Exactly as you said.

---

# 5. The final corrected objective

A practical combined formulation might look like this:

### World model

[
\mathcal{L}_{world}
===================

\mathbb{E}[-\log p(x|z)]
+
\beta KL(q(z|x)||p(z))
+
\gamma \mathcal{L}_{dynamics}
]

---

### Policy + rollout controller

The controller decides **whether to think more**.

[
\pi_{compute}(z_t) \rightarrow k
]

where (k) = number of internal simulation steps.

The objective:

[
\max
\Big(
\mathbb{E}[R]
-------------

C_r \cdot k
\Big)
]

subject to rollout improving value:

[
k^* =
\arg\max_k
\Big(
V_k(z_t) - V_0(z_t) - C_r k
\Big)
]

# 6. Why this avoids your failure case

Now the system behaves differently:

| Situation                 | Compression | Compute allocation |
| ------------------------- | ----------- | ------------------ |
| Predictable trivial state | high        | low compute        |
| Rare high-impact state    | low         | high compute       |
| Random noise              | low         | **low compute**    |

The third row is crucial.

Noise is **uncompressible but useless**.

VoI filters it out.

# 9. One more subtle problem (even deeper)

Even VoI alone is not sufficient.

You also need **uncertainty calibration**:

[
VoI \propto
\sigma^2(V(z'))
]

because if the model is **overconfident**, it will never think.

This is why modern systems use:

* **ensembles**
* **dropout uncertainty**
* **Bayesian world models**
